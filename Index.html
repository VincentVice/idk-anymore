Planning lanes

Organize your work into lanes. Rename or repurpose them as your site grows.

Content lane

  • List your pages and what each must answer.
  • Draft headlines and key messages.
  • Note any media you need (images, video, diagrams).

Design lane

  • Pick color tokens and typography.
  • Sketch desktop and mobile layouts.
  • Decide on navigation and footer structure.

Technical lane

  • Choose hosting (GitHub Pages, etc.).
  • Plan build tools, if any.
  • List integrations (forms, analytics, APIs).

Simple layout patterns

Swap these sections in or out as you design different pages.

Landing page

Hero + key benefits + social proof + clear call‑to‑action.

Docs / knowledge

Sidebar navigation + content pane + search or quick links.

Portfolio

Filterable grid of work + detail pages for case studies.

Blog

Chronological list of posts + categories or tags.
